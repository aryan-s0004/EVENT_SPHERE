SECTION 2 — BUGS FOUND (root-cause analysis)
────────────────────────────────────────────────────────────────────────────────

  BUG #1  [CRITICAL] — GET /api/events returns 404
  ──────────────────────────────────────────────────────────────────────────────
  Symptom  : Events page shows "Unable to load events". Homepage likely blank.
  Root cause: Vercel's [...path] catch-all pattern only matches paths with ≥1
              captured URL segment.  The URL /api/events has ZERO sub-segments,
              so Vercel looks for api/events.js or api/events/index.js — neither
              existed.  Vercel returns 404 before the serverless function is
              ever invoked.
  Introduced by: commit 60f3050 "merge index.js into catch-all handlers"
              which deleted api/events/index.js and moved its logic into
              [...path].js.  The logic is correct; the routing is the bug.
  Affected routes:
    • GET  /api/events          ← events listing (Events.jsx)
    • POST /api/events          ← event creation (AdminDashboard.jsx)
  Status   : FIXED — see Section 3

  BUG #2  [CRITICAL] — POST /api/bookings returns 404
  ──────────────────────────────────────────────────────────────────────────────
  Symptom  : "Book 1 Seat" and "Book Now" buttons fail. Booking creation broken.
  Root cause: Identical issue to BUG #1.  /api/bookings (zero sub-segments)
              has no index handler.  api/bookings/[...path].js is never invoked.
  Affected routes:
    • POST /api/bookings         ← create booking (Events.jsx, EventDetail.jsx)
  Status   : FIXED — see Section 3

  BUG #3  [IMPORTANT] — Missing Access-Control-Allow-Origin header
  ──────────────────────────────────────────────────────────────────────────────
  Symptom  : API calls from external origins (other domains, Postman with CORS
             checks, local dev on different port) fail CORS preflight.
  Root cause: vercel.json defined Access-Control-Allow-Methods and
              Access-Control-Allow-Headers but omitted Access-Control-Allow-Origin.
              Without this header CORS preflight fails entirely.
  Note     : Does NOT affect production same-origin calls (frontend + API on
             same Vercel domain) because same-origin requests skip CORS.
             Affects: local dev on separate ports, external API consumers,
             future mobile app clients.
  Status   : FIXED — see Section 3

  BUG #4  [CONFIG] — GOOGLE_CLIENT_SECRET is empty
  ──────────────────────────────────────────────────────────────────────────────
  Symptom  : Incomplete Google OAuth configuration in .env
  Root cause: .env has GOOGLE_CLIENT_SECRET= (no value set)
  Impact   : The current Google login flow uses One Tap / verifyIdToken() which
             needs only GOOGLE_CLIENT_ID — the secret is NOT used by the
             serverless code.  Therefore Google login currently works despite
             the missing secret.
             If server-side OAuth code exchange is added later, this must be set.
  Status   : NO CODE CHANGE needed — must be set in Vercel Dashboard env vars.
             See Section 5 for instructions.

SECTION 9 — KNOWN LIMITATIONS (not blocking production)
────────────────────────────────────────────────────────────────────────────────

  1. OPTIONS preflight for cross-origin callers:
     The API functions return 405 for OPTIONS requests.  Since the production
     frontend is same-origin with the API (same Vercel deployment), browser CORS
     preflight never fires in production.  External API callers (other domains)
     will fail preflight.  Fix: add an OPTIONS method check in each handler and
     return 204 with empty body.

  2. Image storage as base64 data URIs:
     Event images are stored as base64 strings in MongoDB (≤2MB per image).
     This works but is inefficient at scale.  Future improvement: use Cloudinary
     or Vercel Blob Storage and store image URLs instead.

  3. OTP security:
     OTPs are 6-digit numeric codes with a 10-minute expiry.  A proper
     production system might use shorter expiry or additional rate limiting.
     The 1-minute resend cooldown is in place.

  4. Google sign-in requires origin registration:
     The GOOGLE_CLIENT_ID must have the production Vercel URL added as an
     authorized JavaScript origin in Google Cloud Console.  Without this,
     Google One Tap shows "origin not allowed" and Google login fails silently.

  5. Email App Password required for Gmail:
     EMAIL_PASS must be a Gmail App Password (16-char code), NOT your normal
     Gmail password.  2FA must be enabled on the Gmail account.
